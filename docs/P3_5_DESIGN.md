# P3-5 Context Engine 补强设计

生成时间：2026-07-02

## 目标

让 AIOS 的 Context Pack 从“能用”升级到“更稳、更相关、更适合自动执行”。

本阶段重点补 3 件事：

1. 分层 Pack
2. 更准的相关文件筛选
3. Pack 质量校验

## 范围

本阶段实现：

- Pack 分成任务层 / 项目层 / 文件层
- 任务依赖关系写入 Pack
- 相关文件筛选同时考虑：
  - 任务类型
  - 任务关键词
  - git 变更
  - 文件重要度
- Pack 生成结果返回：
  - `warnings`
  - `quality`
  - `relevant_files`
- Web API 返回 Pack 质量信息

本阶段不实现：

- AST / 符号级检索
- 向量检索
- 跨仓库上下文拼装
- 自动裁剪超长 Pack 为多份子 Pack

## 分层结构

Pack 结构调整为：

1. `任务层`
   - 任务 ID / 标题 / 类型 / 模型
   - 父任务
   - 依赖任务
   - 路由理由
   - 验收标准
   - 风险与校验

2. `项目层`
   - 规则
   - 项目背景
   - 架构说明
   - 架构决策

3. `文件层`
   - 相关文件列表
   - 文件筛选说明

## 文件筛选规则

文件筛选不再只按任务类型做粗过滤，而是组合评分：

- 任务类型匹配
- 路径或摘要命中任务关键词
- git 状态优先
- 重要度优先

并补了常见中英别名，例如：

- `登录 -> login / auth`
- `后台 -> admin / dashboard`
- `接口 -> api / endpoint`

## 质量校验

新增两级质量信息：

### 1. `warnings`

所有发现的提示都会进入 `warnings`，例如：

- 项目还没扫描
- 没选到相关文件
- 项目背景还是占位内容
- 架构说明还是占位内容
- 上下文窗口超过 90%

### 2. `quality`

`quality` 只用于表达“是否有强告警”：

- `ok`
- `warning`

当前强告警包括：

- 没有相关文件
- 验收标准缺失
- 上下文窗口超过 90%

占位内容、未扫描等属于普通提醒，不直接阻断执行。

## CLI / API 变化

### CLI

`aios pack` 现在额外输出：

- `Quality`
- `Relevant files`

### API

`POST /api/pack` 现在额外返回：

- `warnings`
- `quality`
- `relevant_file_count`

## 测试计划

### 自动化测试

1. Pack 内容包含三层结构
2. 关键词文件优先于无关高重要文件
3. git 变更文件优先
4. 占位内容进入 `warnings`
5. 非强告警时 `quality = ok`
6. Web API 返回 `warnings` / `quality`

### 手工验收

1. 创建一个复杂任务
2. 执行 `aios scan`
3. 执行 `aios pack TASK-ID --model gpt-5.5`
4. 打开生成的 Pack
5. 检查是否清楚分成任务层 / 项目层 / 文件层
6. 检查相关文件是否更贴近当前任务
7. 检查质量提示是否合理
