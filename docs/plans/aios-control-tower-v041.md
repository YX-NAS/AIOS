# AIOS 总控台强化方案 v0.41

## 目标

把 launcher 从“多项目入口页”提升为“生产总控台首页”，解决三个核心问题：

1. 进入系统后不知道先处理什么。
2. 自动流程卡住后，只能逐个项目排查。
3. 模型、执行器、索引、目录状态分散，无法快速监控。

本版保持单项目工作区不变，只强化多项目首页的数据聚合、展示方式和日常使用路径。

## 设计原则

- 只做总控，不侵入单项目内部逻辑。
- 首页优先展示“下一步动作”，不是只展示静态统计。
- 数据来源统一由后端聚合，前端只负责渲染。
- 保持 5 秒轮询刷新，避免引入 websocket 复杂度。
- 每个项目仍然独立 `.aios/`，多项目之间不串数据。

## 首页结构

### 1. 今日总览

保留全局统计：

- 项目数
- 生产接入进度
- 今日任务
- 可执行
- 阻塞
- 失败
- 执行中
- 待接管

并继续保留“今日优先查看”项目列表。

### 2. 行动队列

新增四个总控台主视图：

- **立即执行**
  - 展示当前已经满足依赖、预算、模型和 Pack 条件的任务
  - 目标是让用户打开系统后第一时间知道哪些任务可以直接开工

- **待接管**
  - 展示自动化无法继续推进的任务
  - 包括 takeover 队列中的失败分类、原因、建议动作

- **待验收**
  - 展示已经完成执行、等待人工 review / finish 的任务
  - 用于把“代码已产出但未收口”的任务集中出来

- **执行中**
  - 展示当前 running 的任务、关联模型和最近状态
  - 用于判断系统当前是否已有执行链在运行

每条记录至少包含：

- 项目名
- 任务名
- 当前状态
- 推荐或实际模型
- 下一步动作
- 最近更新时间
- 进入项目按钮

### 3. 基础设施状态

新增跨项目基础设施风险看板，聚焦：

- 路径失效
- 待初始化
- 待扫描
- 模型未就绪项目
- 执行器未就绪项目
- Provider 握手失败
- API 权限失败

目标是把“为什么今天跑不动”前置到首页，而不是等进项目后才发现。

### 4. 最近活动

新增跨项目活动流，聚合：

- 最近任务更新
- 最近执行更新
- 最近接管事件

用于快速判断最近哪些项目发生了变化，避免逐个打开检查。

## 后端聚合设计

总控台继续走 `GET /api/workbench`，但返回结构增强：

```json
{
  "project_count": 0,
  "ready_task_count": 0,
  "pending_takeover_count": 0,
  "focus_projects": [],
  "actionable_ready": [],
  "pending_takeovers": [],
  "review_queue": [],
  "active_runs": [],
  "infra_summary": {
    "alerts": []
  },
  "recent_activity": []
}
```

数据来源：

- 任务和优先级：`tasks.json`
- 调度状态：`scheduler_summary()`
- 执行状态：`executions.json`
- 接管状态：`takeover.json`
- 模型就绪状态：全局模型库运行时摘要
- 执行器可用状态：执行器摘要

## 简化使用流程

新的推荐流程：

1. 打开 launcher 首页
2. 先看“待接管”和“基础设施状态”，排除阻塞
3. 再看“立即执行”，决定今天先推进哪条任务
4. 点“进入项目”进入对应单项目工作区
5. 在项目内继续执行、验收、回写
6. 回到总控台观察“待验收”“执行中”“最近活动”是否刷新

这样首页就承担三件事：

- 决定今天先做什么
- 发现哪里卡住了
- 观察多项目是否在正常运行

## 测试要求

自动化测试至少覆盖：

1. `/api/workbench` 返回新的行动队列字段
2. ready / running / takeover 状态能正确聚合到首页摘要
3. 基础设施摘要能返回 alert 列表
4. 最近活动列表在有任务/执行/接管变化时返回记录

人工验收至少覆盖：

1. 首页能看到四类行动队列
2. 点击条目可进入对应项目
3. 首页刷新后队列和活动流会同步变化
4. 接入多个项目时，状态互不串扰

## 落地边界

本版不做：

- 首页直接执行任务
- 首页内嵌单项目控制台
- websocket 实时推送
- 跨项目任务编辑
- 自动调度全局编排

这些能力后续可以继续叠加，但当前版本先把“总控台可看、可判断、可跳转”做好。
